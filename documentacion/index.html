DIAGNÓSTICO

5 problemas críticos

Credenciales en GitHub
Usuario, host y password de la BD de producción están versionados en class/config.php.
CRÍTICO
SQL Injection vía cookies
El código concatena $_COOKIE en queries — un atacante edita su cookie y entra a otra empresa.
CRÍTICO
Sesión sin firmar
La identidad del usuario viaja en cookies cliente sin firma criptográfica ni $_SESSION.
CRÍTICO
Módulos duplicados/legacy
ficha, fichaAE, fichaContacto, _acceso, __agenda… código abandonado conviviendo con el vivo.
ALTO
Sin tests, CI ni documentación
0 pruebas automatizadas, 0 README, 0 pipelines. Cada cambio es manual y a ciegas.
